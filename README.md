# Profiles API

[![CI](https://github.com/akoshodi/profiles-api/actions/workflows/ci.yml/badge.svg)](https://github.com/akoshodi/profiles-api/actions/workflows/ci.yml)
[![PHP](https://img.shields.io/badge/PHP-8.4-777BB4?logo=php)](https://php.net)
[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)

A demographic intelligence REST API built on Slim PHP 4. Accepts names, enriches them with gender, age, and nationality predictions via external classification APIs, and persists profiles to SQLite. Exposes endpoints for filtering, pagination, natural-language search, and a full GitHub OAuth2 + PKCE authentication layer with role-based access control.

**Live base URL:** `https://profiles-api.duckdns.org`

## API Reference

### Authentication Endpoints

All auth endpoints are public (no token required).

#### `GET /auth/github`

Initiates the GitHub OAuth2 + PKCE flow.

| Query param | Required | Description |
|---|---|---|
| `mode` | Yes | `web` (browser flow) or `cli` (PKCE exchange flow) |
| `state` | CLI only | CSRF nonce generated by the CLI |
| `code_challenge` | CLI only | SHA-256 PKCE challenge |

**CLI response — `200 OK`**
```json
{
  "authorize_url": "https://github.com/login/oauth/authorize?...",
  "state": "<state>",
  "code_verifier": "<verifier>"
}
```

**Web response** — redirects the browser to GitHub.

---

#### `GET /auth/github/callback`

GitHub redirects here after the user authorises. Exchanges the code, creates or updates the user record, and issues tokens.

**Web** — sets `access_token`, `refresh_token`, and `csrf_token` as HttpOnly cookies and redirects to the portal.  
**CLI** — returns tokens in the JSON body.

---

#### `POST /auth/github/exchange` *(CLI only)*

Completes the PKCE exchange from the CLI.

```json
{
  "code": "...",
  "state": "...",
  "code_verifier": "..."
}
```

**Success — `200 OK`**
```json
{
  "access_token": "...",
  "refresh_token": "...",
  "user": { "id": "...", "username": "...", "email": "...", "role": "analyst" }
}
```

---

#### `POST /auth/token/refresh`

Rotates the refresh token. The old refresh token is revoked; a new access+refresh pair is returned.

**Bearer / cookie auth required.**

---

#### `POST /auth/logout`

Revokes all active tokens for the current session.

**Bearer / cookie auth required.**

### Profile Endpoints

All profile endpoints require `X-API-Version: 1`. Auth required (see [RBAC](#role-based-access-control)).

#### `POST /api/profiles`

Create a profile by name. Calls all three external APIs and persists the result. Idempotent — submitting the same name returns the existing record.

**Request body**
```json
{ "name": "ella" }
```

**Success — `201 Created`**
```json
{
  "status": "success",
  "data": {
    "id": "019571a2-1c3b-7a4e-9f2d-3b8c1e5d7f90",
    "name": "ella",
    "gender": "female",
    "gender_probability": 0.99,
    "age": 46,
    "age_group": "adult",
    "country_id": "DK",
    "country_name": "Denmark",
    "country_probability": 0.85,
    "created_at": "2026-04-13T10:00:00Z"
  }
}
```

---

#### `GET /api/profiles`

List profiles with filtering, sorting, and pagination.

| Parameter | Type | Description |
|---|---|---|
| `gender` | string | `male` or `female` |
| `age_group` | string | `child`, `teenager`, `adult`, `senior` |
| `country_id` | string | ISO 3166-1 alpha-2 code (e.g. `NG`) |
| `min_age` / `max_age` | integer | Inclusive age range |
| `min_gender_probability` | float | Minimum gender confidence |
| `min_country_probability` | float | Minimum nationality confidence |
| `sort_by` | string | `age`, `created_at`, `gender_probability` (default: `created_at`) |
| `order` | string | `asc` or `desc` (default: `desc`) |
| `page` | integer | Page number (default: `1`) |
| `limit` | integer | Results per page, max `50` (default: `10`) |

**Success — `200 OK`**
```json
{
  "status": "success",
  "page": 1,
  "limit": 10,
  "total": 284,
  "total_pages": 29,
  "links": {
    "self": "/api/profiles?page=1&limit=10",
    "next": "/api/profiles?page=2&limit=10",
    "prev": null
  },
  "data": [{ "..." }]
}
```

---

#### `GET /api/profiles/search`

Natural-language query endpoint. Parses plain English into database filters.

| Parameter | Type | Description |
|---|---|---|
| `q` | string | Plain English query (required) |
| `page` / `limit` | integer | Same pagination as `GET /api/profiles` |

```
GET /api/profiles/search?q=young+males+from+nigeria&page=1&limit=10
```

See [Natural Language Parsing](#natural-language-parsing) for supported syntax.

---

#### `GET /api/profiles/{id}`

Retrieve a single profile by UUID.

---

#### `DELETE /api/profiles/{id}`

Delete a profile by UUID. Admin only.

**Success — `204 No Content`**

---

#### `GET /api/profiles/export`

Download all profiles matching current filters as a CSV file.

---

### Admin Endpoints

Require `admin` role.

#### `GET /api/admin/users`

List all registered users with their roles and status.

#### `PATCH /api/admin/users/{id}/role`

Assign a role to a user.

```json
{ "role": "analyst" }
```

#### `DELETE /api/admin/users/{id}`

Deactivate a user account.

## Authentication Flows

### CLI (PKCE)

```
1. CLI generates: state, code_verifier, code_challenge = SHA256(code_verifier)
2. GET /auth/github?mode=cli&state=<s>&code_challenge=<c>
   → returns authorize_url + code_verifier
3. CLI opens authorize_url in browser; starts localhost:8888/callback listener
4. GitHub redirects to localhost:8888/callback with ?code=...&state=...
5. CLI calls POST /auth/github/exchange {code, state, code_verifier}
6. Backend verifies PKCE, issues access + refresh tokens
7. CLI writes tokens to ~/.insighta/credentials.json (mode 0600)
```

### Browser (HttpOnly Cookies)

```
1. User clicks "Continue with GitHub"
2. GET /auth/github?mode=web  →  browser redirected to GitHub
3. GitHub redirects to /auth/github/callback?code=...&state=...
4. Backend sets cookies:
   - access_token   (HttpOnly, 3 min)
   - refresh_token  (HttpOnly, 5 min)
   - csrf_token     (JS-readable, 5 min)
5. Portal includes X-CSRF-Token header on all mutating requests
```

## Token Lifecycle

| Token | Expiry | Storage |
|---|---|---|
| Access token | 3 minutes | Bearer header / `access_token` cookie |
| Refresh token | 5 minutes | `refresh_token` cookie / CLI file |

- Refresh tokens are single-use. Each `POST /auth/token/refresh` revokes the old token and issues a new pair.
- If the refresh token is expired the user must log in again.
- The CLI auto-refreshes the access token transparently before expiry.

## Role-Based Access Control

| Endpoint | `admin` | `analyst` |
|---|---|---|
| `POST /api/profiles` | ✅ | ❌ |
| `GET /api/profiles` | ✅ | ✅ |
| `DELETE /api/profiles/{id}` | ✅ | ❌ |
| `GET /api/profiles/export` | ✅ | ✅ |
| `GET /api/admin/users` | ✅ | ❌ |
| `PATCH /api/admin/users/{id}/role` | ✅ | ❌ |
| `DELETE /api/admin/users/{id}` | ✅ | ❌ |

The admin account (`ADMIN_EMAIL`) is seeded automatically on first boot. All other users are created with the `analyst` role when they first log in via GitHub OAuth, and the admin can then promote them.

## Natural Language Parsing

`GET /api/profiles/search` uses a rule-based regex parser — no AI or LLMs.

### Supported patterns

**Gender**

| Query | Filter |
|---|---|
| `male`, `men`, `man` | `gender=male` |
| `female`, `women`, `woman`, `girl`, `girls` | `gender=female` |

**Age group**

| Query | Filter |
|---|---|
| `child`, `children`, `kids` | `age_group=child` |
| `teen`, `teenager`, `teenagers`, `adolescents` | `age_group=teenager` |
| `adult`, `adults` | `age_group=adult` |
| `senior`, `seniors`, `elderly`, `old people` | `age_group=senior` |

**Age range**

| Query pattern | Filter |
|---|---|
| `young` | `min_age=16`, `max_age=24` |
| `above X` / `over X` / `older than X` | `min_age=X` |
| `below X` / `under X` / `younger than X` | `max_age=X` |
| `between X and Y` | `min_age=X`, `max_age=Y` |
| `aged X` / `age X` | `min_age=X`, `max_age=X` |

**Country** — full names matched longest-first (e.g. `south africa` before `africa`). Bare ISO codes after `from`/`in` also accepted.

### Example mappings

| Query | Filters |
|---|---|
| `young males from nigeria` | `gender=male, min_age=16, max_age=24, country_id=NG` |
| `females above 30` | `gender=female, min_age=30` |
| `adult males from kenya` | `gender=male, age_group=adult, country_id=KE` |
| `seniors in south africa` | `age_group=senior, country_id=ZA` |
| `women between 20 and 35` | `gender=female, min_age=20, max_age=35` |

### Known limitations

- No negation (`not from nigeria`, `excluding males`)
- OR logic not supported (only AND across all filters)
- Only the first matched country is used
- Adjective forms (`nigerian`, `kenyan`) not mapped — use noun forms
- Name-based search not supported

## Database Seeding

The database ships with 2026 pre-generated profiles. Seeding is idempotent.

```bash
php scripts/seed.php
```

The admin user defined by `ADMIN_EMAIL` is seeded automatically when the app boots for the first time.

## Deployment

Hosted on Ubuntu with Nginx + PHP 8.4-FPM, SSL via Let's Encrypt, DNS via DuckDNS.

```bash
cd /var/www/profiles-api
git fetch origin && git reset --hard origin/main
composer install --no-dev --optimize-autoloader
sudo chown -R www-data:www-data .
sudo systemctl restart php8.4-fpm
```
